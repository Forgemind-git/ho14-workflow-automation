What you build

A recurring automation built and scheduled with Claude Code — the automation script, how it's scheduled, a 5-step Loom, and a measured before/after time log. Each sample runs unattended on a schedule (cron, hourly, daily or weekly) so a manual chore disappears for good: it fetches or watches, transforms the data, and stores or sends the result with no human in the loop. You prove the value with a simple time log showing the manual minutes removed.

Pick a problem statement

Each sample is a real, hosted build — the script, its schedule and a sample run all filled in. Open one live, or bring your own recurring chore (get it approved first).

1Daily Web Scraper → Sheet

You copy the same numbers off a website into a tracking sheet every morning and it is soul-crushing. Build a coded automation with Claude Code that scrapes that page daily, cleans the data, and appends it to a sheet or database, scheduled on a cron.

✓ it runs unattended each day and your time log shows the daily minutes saved Open live sample
2API Fetch → Transform → Store

You manually pull from an API, reshape it, and drop it somewhere every day. Build a coded pipeline with Claude Code that fetches from the API, transforms the data, and stores it or sends a notification, triggered on a schedule.

✓ a cron-driven fetch-transform-store loop with a before/after time log Open live sample
3File Drop Watcher & Processor

Files land in a folder and someone has to process each one by hand. Build a file-watcher automation with Claude Code that detects a new file, processes it (convert or extract), and writes the output.

✓ dropping a file triggers the job automatically, with a time log of manual minutes removed Open live sample
4Scheduled Report Emailer

Your boss expects the same report emailed every week and assembling it eats an hour. Build a scheduled job with Claude Code that queries the data, builds the report, and emails it weekly.

✓ the email arrives on schedule with no manual steps — the recurring hour saved Open live sample

Two of your systems drift out of sync and you reconcile them by hand. Build a coded sync job with Claude Code that keeps a sheet, database or tool aligned automatically on a schedule.

Use your Claude.ai Pro plan wisely
Runs on your machine (or your server)
  • This is a scheduled, coded automation you run on your own machine — not on this GitHub Pages page. Pages is static, so it can't run a cron job; it only hosts this landing page and the README.
  • Follow the README to run the script once, then schedule it with cron (or Task Scheduler) so it runs unattended.
  • Want it running on a public server later? The course covers that once — see the Week-6 deploy walkthrough video.
Subscription-first — no API key
  • Everything is built with your normal Claude.ai login (Pro or Team) — no API key needed.
  • Use the example prompt in each sample's README — it's already written and tested. Don't reinvent it.
  • Start a new chat when you switch tasks — long chats re-read every earlier message and use your limit faster.
  • Using Claude Code or Cowork? This repo's CLAUDE.md makes Claude follow these same rules automatically, and SKILL.md is a reusable "token-wise" skill.
