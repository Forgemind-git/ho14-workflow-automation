Hands-On 14 · Sample 03

File Drop Watcher & Processor

Automatically detect new CSV files in a Google Drive folder, compute summaries, and archive processed copies — the moment a file lands.

Subscription required

Claude.ai Pro or Team plan with Cowork access. No API key or local Python setup needed.

What this does

This automation watches an input folder (Google Drive via Cowork) for new CSV files. When one appears, it reads the file, computes a summary (row count, column totals for numeric columns), writes a processed copy with a __processed_at column, and moves the original to an archive folder.

  • Detects new CSV files added to a watched Google Drive folder
  • Reads all rows and identifies numeric columns automatically
  • Computes row count and per-column totals
  • Outputs a processed copy with an added __processed_at timestamp column
  • Moves the original to an archive folder to avoid reprocessing
Subscription required

This guide uses Claude Cowork — a scheduled automation feature available on the Claude.ai Pro and Team plans.

  • Go to claude.ai and sign in
  • You need a Pro or Team subscription
  • Connect your Google Drive in Cowork settings so Claude can read/write files
Step-by-step in Claude Cowork
  1. Open claude.ai and click Cowork in the left sidebar.
  2. In Cowork settings, connect your Google Drive account. Create two folders in Drive: input/ (where you drop files) and processed/ (archive destination).
  3. Click New Automation and set the schedule to Every 15 minutes (or hourly if files are less frequent).
  4. In the "What should Claude do?" field, paste the prompt below.
  5. Replace [INPUT_FOLDER_ID] and [ARCHIVE_FOLDER_ID] with the Google Drive folder IDs from your Drive URLs.
  6. In the output settings, choose Save to Google Drive → processed/ folder. Claude will write the processed CSV there.
  7. Click Save & Activate. Drop any CSV into the input/ folder and it will be processed on the next run.
You are a file processing automation. Each time you run, do the following:

1. Check my Google Drive folder [INPUT_FOLDER_ID] for any CSV files that have NOT yet been processed.
   (A file is "processed" if a matching file exists in folder [ARCHIVE_FOLDER_ID])

2. For each unprocessed CSV file found:
   a. Read all rows using the first row as the header.
   b. Count the total number of data rows (excluding header).
   c. Identify all columns that contain only numeric values.
   d. Compute the sum total for each numeric column.
   e. Record the current UTC timestamp as: YYYY-MM-DD HH:MM:SS

3. Build a processed output CSV:
   - Keep all original columns and rows exactly as-is.
   - Add a new column "__processed_at" as the last column, filled with the current timestamp.
   - Add a TOTALS row at the bottom: fill non-numeric columns with "TOTAL", fill numeric columns with their sums.

4. Save the processed CSV to Google Drive folder [ARCHIVE_FOLDER_ID] with filename: processed_[original_filename]

5. Print a summary for each file processed:
   File: [filename]
   Rows: [count]
   Numeric columns: [list with totals]
   Saved as: processed_[filename]

If no new files are found, output only: "No new files to process."

=======================================================

  Rows     : 3
  Columns  : 4 (date, product, quantity, price)
  Totals   : quantity=42, price=1250.00
  Saved as : processed_sales_june.csv

Reference: local Python version
The file main.py in this folder is the original local Python implementation using the watchdog library to monitor a local filesystem directory. It requires pip install -r requirements.txt and runs continuously until Ctrl+C. The Cowork guide above achieves the same result by polling Google Drive on a schedule instead.
