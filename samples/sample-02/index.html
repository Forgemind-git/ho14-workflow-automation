API Fetch → Transform → Store Pipeline

Pull data from any public API on a schedule, rename and reshape it to your schema, and store a clean historical record automatically.

Subscription required: Claude.ai Pro or Team plan with Cowork access. No API key or local Python setup needed.

What this does

This automation runs a full ETL pipeline on a schedule: it fetches JSON from a public API, transforms the data (renames fields, formats values, decodes codes), and stores a clean timestamped record in your chosen output (Google Sheet or CSV).

  • Calls a public API endpoint (demonstrated with Open-Meteo weather — no API key needed)
  • Extracts the fields you care about from the JSON response
  • Renames API field names to readable column names (e.g. weathercode → weather_condition)
  • Inserts a timestamped record into your data store
  • Idempotent: safe to re-run without creating duplicates

Subscription required

This guide uses Claude Cowork — a scheduled automation feature available on the Claude.ai Pro and Team plans.

  • Go to claude.ai and sign in
  • You need a Pro or Team subscription
  • Cowork handles the fetch, transform, and output steps — no local environment needed

Step-by-step in Claude Cowork

  1. Open claude.ai and click Cowork in the left sidebar.
  2. Click New Automation and set the schedule to Hourly or Daily depending on how fresh you need the data.
  3. In the "What should Claude do?" field, paste the prompt below.
  4. Replace [CITY], [LATITUDE], and [LONGITUDE] with your target location. You can look up coordinates at latlong.net.
  5. In the output settings, choose Append to Google Sheet and name the sheet weather_data. Set the first row as a header row.
  6. Click Save & Activate. Each run will add one new row to your sheet.
You are an API pipeline automation. Each time you run, do the following:

1. Fetch this URL (Open-Meteo public API, no key required):
   https://api.open-meteo.com/v1/forecast?latitude=[LATITUDE]&longitude=[LONGITUDE]¤t_weather=true

2. Parse the JSON response and extract these fields from current_weather:
   - temperature (°C)
   - windspeed (km/h)
   - winddirection (degrees)
   - weathercode (integer)
   - time (ISO datetime string)

3. Transform the data:
   - Rename "time" → "recorded_at"
   - Rename "weathercode" → "weather_code"
   - Add a "city" column with value: [CITY]
   - Add a "fetched_at" column with the current UTC timestamp (YYYY-MM-DD HH:MM:SS)
   - Decode weather_code to a human-readable condition using WMO code table
     (e.g. 0=Clear sky, 1=Mainly clear, 2=Partly cloudy, 3=Overcast,
      45=Fog, 61=Light rain, 63=Moderate rain, 80=Slight showers, etc.)
   - Add a "weather_condition" column with the decoded text

4. Output ONE CSV row (no header) in this column order:
   fetched_at,city,recorded_at,temperature,windspeed,winddirection,weather_code,weather_condition

5. After the CSV row, print a one-line summary:
   "Pipeline complete: [CITY] at [recorded_at] — [temperature]°C, [weather_condition]"

Output only the CSV row and the summary line. No other text.

Reference: local Python version

The file main.py in this folder is the original local Python implementation. It requires pip install -r requirements.txt, a .env file with coordinates, and writes to a local SQLite database. The Cowork guide above achieves the same ETL result without any local setup.
