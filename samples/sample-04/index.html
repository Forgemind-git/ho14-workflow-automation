Scheduled Report Emailer

Read a metrics data source every week, build a formatted HTML email report with totals and trends, and send it automatically to your team.

Subscription required: Claude.ai Pro or Team plan with Cowork access. No API key, SMTP credentials, or local Python setup needed.

What this does

This automation reads your metrics data source on a weekly schedule, computes column totals and key highlights, builds a professionally formatted HTML email report, and sends it to your team's inboxes — every Monday morning before anyone starts their day.

  • Reads the latest data from a Google Sheet or uploaded CSV
  • Computes totals for all numeric columns
  • Identifies the top-performing row and any notable outliers
  • Builds a styled HTML email with a data table, summary section, and totals row
  • Sends via Claude Cowork's built-in email integration (no SMTP setup required)
  • Logs each send so you have a record of what was sent and when

Subscription required

This guide uses Claude Cowork — a scheduled automation feature available on the Claude.ai Pro and Team plans.

  • Go to claude.ai and sign in
  • You need a Pro or Team subscription
  • Cowork can send emails through your connected Gmail or Outlook account — no SMTP credentials required

Step-by-step in Claude Cowork

  1. Open claude.ai and click Cowork in the left sidebar.
  2. In Cowork settings, connect your Gmail or Outlook account so Claude can send emails on your behalf.
  3. Connect your Google Drive or Sheets account, then share the metrics spreadsheet with Claude (paste the Sheet URL or Drive file ID).
  4. Click New Automation and set the schedule to Weekly — pick Monday at 07:00 so the report arrives before the work day starts.
  5. Paste the prompt below, replacing the placeholders with your spreadsheet ID and recipient addresses.
  6. Click Save & Activate. Each Monday, Claude will read your data, build the report, and email it automatically.
You are a weekly report automation. Each time you run, do the following:

1. Read the data from my Google Sheet: [SHEET_ID_OR_URL]
   Use the first row as column headers.

2. Analyse the data:
   a. Count the total number of data rows.
   b. For every numeric column, compute: sum, average, min, max.
   c. Find the row with the highest value in the primary metric column (the first numeric column).
   d. Note any values more than 2x the column average (flag as "high").

3. Build an HTML email report with these sections:
   HEADER: "Weekly Metrics Report — Week of [Monday's date]"
   SUMMARY: Bullet list — total rows, primary metric total, top row highlight
   DATA TABLE: All rows styled as an HTML table with alternating row colours (#f9fafb / #ffffff), a header row in #0E7490 with white text, and a TOTALS footer row in bold
   INSIGHTS: 2–3 plain-English observations based on the data (trends, outliers, notable changes)
   FOOTER: "Generated by Claude Cowork · [current date]"

4. Send the email via my connected email account:
   To: [RECIPIENT_EMAILS]
   Subject: "Weekly Metrics Report — [current date]"
   Body: the HTML report above

5. Output a one-line confirmation:
   "Report sent to [recipients] at [timestamp]. Rows: [count]. Primary metric total: [value]"

Reference: local Python version

The file main.py in this folder is the original local Python implementation. It requires pip install -r requirements.txt, a .env file with Gmail app-password SMTP credentials, and a cron job. The Cowork guide above sends email through your connected account with no SMTP configuration needed.
